# WORK ORDER 0001

## Title

MachineDex Visual Identity Foundation

Created:
July 3, 2026

Status:
In Progress

---

## Objective

Replace the generic prototype styling with the official MachineDex visual language.

This work order is focused entirely on establishing the visual identity of the application.

This is **not** a functionality milestone.

The application should immediately feel like MachineDex before any additional features are developed.

---

## Project References

Use the following files as the primary design authority.

docs/Design/StyleGuide.md

MachineDex Vision image located within docs/Design/

The Style Guide and Vision image override the appearance of the current application.

---

## Design Philosophy

MachineDex should feel like an engineering field manual.

Design adjectives:

• Professional
• Industrial
• Technical
• Purpose-built
• Military simplicity
• Blueprint inspired
• Clean
• Confident

Avoid:

• Generic SaaS dashboards
• Cyberpunk
• Glassmorphism
• Rounded consumer UI
• Bright neon aesthetics

---

## Visual Goals

Create a visual identity that immediately communicates:

"This is a professional engineering reference."

The interface should feel appropriate for a precision manufacturing company.

---

## Color Palette

Primary

- Deep Navy

Secondary

- Warm White

Accent

- Brass

Supporting

- Steel Gray

Background

- Blueprint Navy

---

## Typography

Large bold machine names.

Technical condensed labels.

High information density.

Excellent readability.

---

## Layout

Desktop first.

Three-column layout.

LEFT

Navigation

CENTER

Machine browser

RIGHT

Machine information panel

This layout should become the permanent foundation of the application.

---

## Machine Cards

Machine cards should become the primary visual element.

Each card should display:

Illustration

Machine Name

MDX ID

Machine Type

Operational Status

Cards should resemble pages from a technical equipment manual.

---

## Illustration Integration

Reserve a prominent area for pen-and-ink machine illustrations.

If the current artwork cannot yet be automatically loaded, create the layout so artwork can be added later without redesigning the interface.

---

## Data

Continue using the current static machine dataset.

No backend.

No database.

No maintenance system.

No dynamic parsing.

This milestone is about presentation only.

---

## Code Quality

Create reusable components.

Avoid duplicated styling.

Keep the project organized for future expansion.

---

## Deliverable

When the application starts with:

npm run dev

The interface should immediately look substantially closer to the MachineDex Vision image than the current prototype.

The visual identity should feel unique and recognizable.

Someone seeing Version 2 should immediately recognize it as MachineDex rather than a generic React application.
